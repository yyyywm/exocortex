# 合作式调度

## 直觉（Intuition）

合作式调度就像一群人在一个房间里轮流发言：每个人说完自己的部分后，主动把话筒递给下一个人。如果某个人霸占话筒不放，所有人就得等着。

与抢占式调度（Preemptive）不同，合作式调度中任务自己决定何时让出 CPU。

## 形式化定义（Definition）

合作式调度（Cooperative Scheduling）是一种任务调度策略，其中：

- 每个任务在执行时拥有完整的 CPU 控制权
- 任务必须主动 yield（让出）CPU，调度器才会切换到下一个任务
- 没有外部强制中断机制

## 关键代码

```c
// 典型的合作式调度主循环
while (true) {
    task_A();  // 执行完才返回
    task_B();  // 执行完才返回
    task_C();  // 执行完才返回
}
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 实现简单，无需上下文切换开销 | 一个任务卡死，整个系统卡住 |
| 无需复杂的同步机制 | 实时性无法保证 |
| 确定性高（执行顺序可预测） | 任务必须自律，不能长时间占用 CPU |
