# 发布规范

## 1. 技术栈

| 组件 | 版本 | 用途 |
|------|------|------|
| Quartz | 4.x | 静态站点生成器 |
| Node.js | >= 20 | Quartz 运行环境 |
| GitHub Pages | - | 免费静态托管 |
| GitHub Actions | - | 自动化构建与部署 |

## 2. Quartz 配置要点

### 内容目录

Quartz 读取 `content/` 作为笔记源目录。

### 忽略规则（Ignore Patterns）

在 `quartz.config.ts` 的 `configuration` 中配置：

```typescript
ignorePatterns: [
  "00-Inbox",
  "sources",
  "templates",
  "_system",
  ".obsidian",
  "80-Blog/drafts",
],
```

**规则说明**：
- `00-Inbox`：碎片永不发布
- `sources`：源码仓库不发布
- `templates`：模板不发布
- `_system`：系统规范不发布
- `.obsidian`：Obsidian 配置不发布
- `80-Blog/drafts`：博客草稿不发布

### 发布过滤（Draft 控制）

在 `quartz.config.ts` 的 `plugins.filters` 中配置：

```typescript
filters: [
  Plugin.RemoveDrafts(),  // 移除 frontmatter 中 draft: true 的笔记
],
```

**双重控制**：
1. **目录级**：`ignorePatterns` 硬排除整个目录
2. **文件级**：`RemoveDrafts()` 软排除单篇笔记

### 主页配置

- **主页文件**：`content/index.md`
- **内容要求**：个人简介 + 数字花园导航 + 最近更新
- **Frontmatter**：
  ```yaml
  ---
  title: "Digital Garden"
  description: "个人知识库与数字花园"
  draft: false
  ---
  ```

## 3. 发布控制策略

### 单篇笔记控制

通过 frontmatter 控制：

```yaml
---
draft: true   # 不发布
draft: false  # 发布（默认值）
---
```

### 目录级控制

| 目录 | 默认行为 | 说明 |
|------|----------|------|
| `content/00-Inbox/` | 永不发布 | 硬排除 |
| `content/80-Blog/drafts/` | 永不发布 | 硬排除 |
| `content/80-Blog/posts/` | 默认发布 | 可单篇 draft: true 隐藏 |
| `content/90-MOCs/` | 默认发布 | 导航页 |
| `content/70-Projects/` | 默认发布 | 项目展示 |
| `content/10-Atoms/` | 默认发布 | 可单篇 draft: true 隐藏 |

### 博客文章专用 frontmatter

博客文章（`80-Blog/posts/` 下）必须包含：

```yaml
---
title: "文章标题"
type: blog
description: "文章摘要，用于 SEO 和列表展示"
tags: ["标签1", "标签2"]
date: "2026-06-06"
draft: false
---
```

## 4. 构建与部署流程

### 本地预览

```bash
# 安装依赖（首次）
npm install

# 本地构建并预览
npx quartz build --serve
```

访问 `http://localhost:8080` 预览。

### 生产部署（自动）

由 GitHub Actions 自动执行：

```yaml
# .github/workflows/deploy.yml
name: Deploy Quartz site to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:  # 允许手动触发

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install Dependencies
        run: npm ci

      - name: Build Quartz
        run: npx quartz build

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./public

  deploy:
    needs: build
    runs-on: ubuntu-latest
    permissions:
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### 部署触发条件

1. `git push origin main`
2. GitHub 仓库页面 → Actions → Deploy Quartz site → Run workflow

## 5. URL 结构

### 默认路径

| 笔记路径 | 网站 URL |
|----------|----------|
| `content/index.md` | `/` |
| `content/10-Atoms/cs/哈希表.md` | `/10-Atoms/cs/哈希表` |
| `content/80-Blog/posts/xxx.md` | `/80-Blog/posts/xxx` |
| `content/90-MOCs/知识地图.md` | `/90-MOCs/知识地图` |

### 自定义 slug（可选）

在 frontmatter 中使用 `permalink`：

```yaml
---
title: "从零理解 ArduPilot 调度器"
permalink: "ardupilot-scheduler"
---
```

生成 URL：`/ardupilot-scheduler`

## 6. 自定义域名（可选）

如需使用自定义域名（如 `notes.yourdomain.com`）：

1. 在 DNS 提供商处添加 CNAME 记录：
   - 主机记录：`notes` 或 `@`
   - 记录值：`yourname.github.io`

2. 在仓库创建 `content/CNAME` 文件：
   ```
   notes.yourdomain.com
   ```

3. 在 GitHub 仓库 Settings → Pages → Custom domain 中填入域名

4. 勾选 "Enforce HTTPS"

## 7. 性能优化

### 构建速度
- 笔记量 < 1000：构建时间 < 30 秒
- 笔记量 > 1000：考虑使用 `ignorePatterns` 排除旧归档

### 图片处理
- 图片放入 `content/` 下与笔记同目录或 `content/_assets/`
- 建议使用 WebP 格式
- 大图（> 1MB）考虑压缩或使用图床

## 8. 故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 网站 404 | GitHub Pages 未启用 | Settings → Pages → Source: GitHub Actions |
| 某些笔记不显示 | 在 ignorePatterns 中 | 检查 `quartz.config.ts` |
| 构建失败 | Node 版本过低 | 确保 Node >= 20 |
| 样式丢失 | baseUrl 配置错误 | 检查 `baseUrl` 是否与仓库名一致 |
| 中文乱码 | 文件编码问题 | 确保所有 `.md` 文件为 UTF-8 |
