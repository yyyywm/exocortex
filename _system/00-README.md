# Digital Garden 系统总览

## 这是什么

本仓库是一个基于 **Obsidian + Quartz + GitHub Pages** 的个人数字花园，同时作为 **AI 增强的知识操作系统（PKOS）**。

它不仅是笔记库，更是一套**可复现、可迁移、可自动化**的个人知识基础设施。

## 重建顺序

如果要从零重建本系统，按以下顺序执行：

1. 阅读 `05-依赖与工具.md`，安装必需软件
2. 阅读 `01-架构规范.md`，理解目录结构
3. 阅读 `06-初始化检查清单.md`，执行初始化
4. 阅读 `03-AI-操作规范.md`，理解 Claude Code 的角色

## 规范文档索引

| 文档 | 用途 |
|------|------|
| `01-架构规范.md` | 人类和 AI 都必须遵守的结构约定 |
| `02-笔记规范.md` | 每条笔记的元数据与内容标准 |
| `03-AI-操作规范.md` | Claude Code 的标准作业程序（SOP） |
| `04-发布规范.md` | 博客发布的技术细节 |
| `05-依赖与工具.md` | 环境复现清单 |
| `06-初始化检查清单.md` | 从零开始的步骤 |

## 系统哲学

- **文档即规范，规范即接口**：所有约定写在 Markdown 里，不在人脑中
- **AI 是规范执行器**：Claude Code 按照 SOP 执行，不是自由创作
- **Git 是操作日志**：所有变更可追溯、可回滚
- **笔记是活的知识资产**：持续连接、持续迭代、持续发布

## 快速入口

- **记笔记**：打开 Obsidian，进入 `content/`
- **问 AI**：在终端 `cd ~/digital-garden && claude`
- **看博客**：访问 `https://yourname.github.io/digital-garden/`
