# 架构规范

## 1. 目录结构（不可随意变更）

| 路径 | 职责 | 发布策略 |
|------|------|----------|
| `content/00-Inbox/` | 碎片捕获，未经处理 | 永不发布 |
| `content/10-Atoms/` | 原子概念，最小知识单元 | `draft: false` 时发布 |
| `content/20-Compounds/` | 复合系统，由原子组装 | `draft: false` 时发布 |
| `content/30-Derivations/` | 数学/逻辑推导 | `draft: false` 时发布 |
| `content/40-Implementations/` | 可运行代码实现 | `draft: false` 时发布 |
| `content/50-Sources/` | 来源精读（论文/书籍/代码） | `draft: false` 时发布 |
| `content/60-Insights/` | 个人独特洞察 | `draft: false` 时发布 |
| `content/70-Projects/` | 项目记录与学习日志 | 默认发布（可 `draft: true` 隐藏） |
| `content/80-Blog/drafts/` | 博客草稿 | 永不发布 |
| `content/80-Blog/posts/` | 已发布博客 | 默认发布 |
| `content/90-MOCs/` | 内容地图与导航 | 默认发布 |
| `sources/` | 外部代码库（Git 子模块） | 永不发布 |
| `_system/` | 系统规范文档 | 永不发布 |
| `templates/` | Obsidian 模板 | 永不发布 |

## 2. 命名规范

### 文件命名
- 使用 **kebab-case**（短横线连接）
- 不包含空格、特殊字符
- 中文标题也使用 kebab-case 罗马化或拼音，或直接用中文短横线
- 示例：`AP-Scheduler.md`、`卡尔曼滤波.md`、`PID-控制器.md`、`zero-to-hero.md`

### 目录命名
- 使用英文小写
- 领域目录：`cs/`, `math/`, `control/`, `embedded/`, `software/`, `tools/`

## 3. 分类体系（Domain 枚举）

`domain` frontmatter 字段必须从以下枚举中选取：

| domain 值 | 中文名 | 说明 |
|-----------|--------|------|
| `cs` | 计算机科学 | 算法、数据结构、体系结构、网络 |
| `math` | 数学 | 线性代数、微积分、概率统计、优化 |
| `control` | 控制理论 | PID、卡尔曼滤波、状态空间、MPC |
| `embedded` | 嵌入式系统 | MCU、RTOS、驱动开发、硬件抽象 |
| `software` | 软件工程 | 设计模式、系统架构、DevOps |
| `tools` | 工具与方法 | Git、Obsidian、AI 工具、工作流方法论 |
| `ardupilot` | ArduPilot | ArduPilot 飞控系统专项 |
| `workflow` | 工作流 | 个人知识管理、学习方法论、效率 |
| `meta` | 元知识 | 关于本系统本身的知识 |

## 4. 笔记类型（Type 枚举）

| type 值 | 说明 | 存放目录 |
|---------|------|----------|
| `atom` | 原子概念，不可再分的最小知识单元 | `content/10-Atoms/{domain}/` |
| `compound` | 复合系统，由多个原子概念组装而成 | `content/20-Compounds/` |
| `derivation` | 数学推导、逻辑证明、公式推导 | `content/30-Derivations/` |
| `implementation` | 代码实现、最小可运行示例 | `content/40-Implementations/` |
| `source` | 来源精读（论文、书籍、代码库） | `content/50-Sources/` |
| `insight` | 个人洞察、独特发现、反思 | `content/60-Insights/` |
| `project` | 项目记录、学习周记、实验记录 | `content/70-Projects/` |
| `blog` | 博客文章，面向公众的 polished content | `content/80-Blog/{drafts|posts}/` |
| `moc` | 内容地图（Map of Content），导航页 | `content/90-MOCs/` |

## 5. 成熟度（Status 枚举）

| status | 说明 |
|--------|------|
| `seedling` | 幼苗，刚创建，内容不完整，待孵化 |
| `budding` | 萌芽，有基本内容，已建立部分连接 |
| `evergreen` | 常青，内容成熟，持续维护，值得反复回顾 |

## 6. 隐私与发布策略

### 三层控制

1. **目录级硬排除**：`ignorePatterns` 中配置的目录永不进入构建
2. **Frontmatter 软控制**：单篇笔记设置 `draft: true` 不发布，`draft: false` 发布
3. **显式发布**：部分高级配置可使用 `publish: true` 显式标记

### 默认规则

- `00-Inbox/`、`80-Blog/drafts/`、`sources/`、`templates/`、`_system/`、`.obsidian/`：**硬排除**
- `80-Blog/posts/`、`90-MOCs/`、`70-Projects/`：**默认发布**（除非 `draft: true`）
- 其他目录：**默认发布**（除非 `draft: true`）

## 7. 扩展预留

新增领域时：
1. 在 `01-架构规范.md` 的 domain 枚举表中添加新行
2. 在 `content/10-Atoms/` 下创建新目录
3. 更新 `03-AI-操作规范.md` 中的决策树（如需要）

新增笔记类型时：
1. 在 type 枚举表中添加新行
2. 在 `templates/` 下创建新模板
3. 在 `03-AI-操作规范.md` 中添加对应 SOP
