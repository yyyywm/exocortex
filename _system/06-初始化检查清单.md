# 初始化检查清单

## 说明

本清单用于从零开始重建 Digital Garden 系统。按顺序执行，每完成一项打勾。

**目标状态**：GitHub Pages 网站可访问 + Claude Code 能执行 SOP + Obsidian 能正常编辑。

---

## Phase 1：环境准备

- [ ] **1.1** 安装 Git（`git --version` >= 2.40）
- [ ] **1.2** 安装 Node.js（`node --version` >= 20.x）
- [ ] **1.3** 安装 Obsidian（[obsidian.md](https://obsidian.md/)）
- [ ] **1.4** 安装 Claude Code：`npm install -g @anthropic-ai/claude-code`
- [ ] **1.5** 配置 Git 用户名和邮箱
  ```bash
  git config --global user.name "Your Name"
  git config --global user.email "your.email@example.com"
  ```
- [ ] **1.6** 配置 GitHub SSH Key（或确认 HTTPS 凭据可用）
- [ ] **1.7** 在 GitHub 创建私有仓库 `digital-garden`

**验证**：`git --version && node --version && claude --version` 全部有输出。

---

## Phase 2：仓库初始化

- [ ] **2.1** 本地创建目录
  ```bash
  mkdir ~/digital-garden && cd ~/digital-garden
  ```
- [ ] **2.2** 初始化 Git
  ```bash
  git init
  git remote add origin git@github.com:yourname/digital-garden.git
  ```
- [ ] **2.3** 创建目录结构（或从已有仓库克隆）
  ```bash
  mkdir -p content/{00-Inbox,10-Atoms/{cs,math,control,embedded,software,tools},20-Compounds,30-Derivations,40-Implementations/ardupilot-traces,50-Sources/{Papers,Books,Courses,Codebases/ArduPilot},60-Insights,70-Projects,80-Blog/{drafts,posts},90-MOCs/System}
  mkdir -p templates
  mkdir -p sources
  mkdir -p _system
  ```

**验证**：`ls -R content/` 能看到完整的目录树。

---

## Phase 3：添加系统规范文档

- [ ] **3.1** 确认 `_system/` 目录下有 7 份规范文档：
  - `00-README.md`
  - `01-架构规范.md`
  - `02-笔记规范.md`
  - `03-AI-操作规范.md`
  - `04-发布规范.md`
  - `05-依赖与工具.md`
  - `06-初始化检查清单.md`
- [ ] **3.2** 确认 `templates/` 目录下有模板文件（见 Phase 6）

**验证**：`ls _system/ && ls templates/`

---

## Phase 4：初始化 Quartz

- [ ] **4.1** 初始化 Quartz 项目
  ```bash
  cd ~/digital-garden
  npx quartz create
  ```
  按提示选择：
  - Content directory: `content`
  - 其他选项按需选择

- [ ] **4.2** 确认生成的文件：
  - `quartz.config.ts`
  - `package.json`
  - `quartz/` 目录

- [ ] **4.3** 根据 `04-发布规范.md` 修改 `quartz.config.ts`：
  - 配置 `ignorePatterns`
  - 配置 `baseUrl`
  - 配置 `filters: [Plugin.RemoveDrafts()]`

- [ ] **4.4** 安装依赖
  ```bash
  npm install
  ```

**验证**：`npx quartz build` 能成功执行，生成 `public/` 目录。

---

## Phase 5：配置 GitHub Actions

- [ ] **5.1** 创建工作流目录
  ```bash
  mkdir -p .github/workflows
  ```
- [ ] **5.2** 创建 `.github/workflows/deploy.yml`（内容见 `04-发布规范.md`）

- [ ] **5.4** 配置仓库基础信息：
  - Settings → General → 勾选 "Issues"（可选，用于知识管理）

**验证**：工作流文件已提交到仓库。

## 常见问题

### Q: `npx quartz create` 报错？
A: 检查 Node.js 版本（`node --version` 需 >= 20）。

### Q: GitHub Actions 构建失败？
A: 检查 `quartz.config.ts` 是否有语法错误，或 `package.json` 是否完整。

### Q: 网站 404？
A: 确认 GitHub Pages 的 Source 设置为 "GitHub Actions"，而不是 "Deploy from a branch"。

### Q: Claude Code 无法读取文件？
A: 确认在 `~/digital-garden` 目录下启动 `claude`，且文件权限正确。

### Q: 子模块拉取失败？
A: ArduPilot 仓库较大，可能需要较长时间。可用 `git submodule update --init --depth 1` 浅克隆。
